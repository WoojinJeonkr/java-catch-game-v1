## ✅ 커밋 컨벤션

팀원 간의 일관된 협업을 위해 아래와 같은 커밋 컨벤션을 사용합니다.

### 📌 커밋 타입

| 타입       | 설명                                 |
|------------|--------------------------------------|
| `feat`     | 새로운 기능 추가                      |
| `fix`      | 버그 수정                             |
| `docs`     | 문서 수정 (README, 주석 등)           |
| `style`    | 코드 스타일 수정 (공백, 세미콜론 등)  |
| `refactor` | 코드 리팩토링 (기능 변화 없음)         |
| `test`     | 테스트 코드 추가/수정                 |
| `chore`    | 설정, 패키지 매니저 등 기타 작업       |
